SAC Inteligente Peruzzo — Referência n8n

Backend MVP para registrar atendimentos, chamados, mídia e satisfação. O n8n orquestra a conversa; esta API guarda o estado estruturado.

Base URL: http://localhost:3000 Content-Type: application/json Auth em todos os endpoints
Autenticação — envie o token de integração em todas as requisições:

Authorization: Bearer <N8N_INTEGRATION_TOKEN>
ou x-integration-token: <N8N_INTEGRATION_TOKEN>

PATCH /sac-attendances/:id Cancelar, fechar ou encerrar sem satisfação

Transições estreitas permitidas no MVP. Sempre envie status alvo.

1 — Cancelar (started ou collecting_data)

{
  "status": "cancelled",
  "cancellationReason": "timeout",
  "lastSummary": "Obrigatório quando cancellationReason = other"
}

Motivos: customer_gave_up, operational_duplicate, invalid_message, spam, timeout, other. Preenche closedAt.

2 — Fechar sem chamado (categoria orientativa)

{
  "status": "fechado",
  "detectedCategory": "INFORMACAO_LOJA",
  "lastSummary": "Cliente recebeu orientação validada."
}

Categorias orientativas: FALAR_RH, FALAR_DP, CURRICULO, FORNECEDOR, INFORMACAO_LOJA. Rejeita se já existir chamado não cancelado.

3 — Fechar sem satisfação (pesquisa_satisfacao)

{
  "status": "fechado",
  "closedWithoutSatisfaction": true
}

Só aceito em pesquisa_satisfacao. Preenche closedAt.

POST /sac-attendances/:attendanceId/satisfaction-response Registrar satisfação

Só aceito com atendimento em pesquisa_satisfacao. Uma resposta por atendimento. Fecha o ciclo (fechado + closedAt).

{
  "problemResolvedByCustomer": true,
  "rating": 5,
  "comment": "Opcional"
}
201Resposta criada, atendimento fechado
409Resposta duplicada
400Status inválido ou payload inválido
POST /sac-cases Criar chamado + protocolo

Só quando atendimento está em started ou collecting_data. Move atendimento para waiting_resolution. Gera protocolo SAC-YYYYMMDD-000001.

{
  "attendanceId": "attendance-id",
  "category": "PRODUTO_ESTRAGADO",
  "description": "Descrição sintetizada pelo n8n (10–2000 chars).",
  "storeId": "store-id",
  "rawStoreMention": "loja da rua x",
  "needsHumanReview": false,
  "missingRequiredFields": [],
  "riskFlag": false,
  "riskReasons": [],
  "markAsSentToDkw": false
}
Categorias que geram chamado: DENUNCIA, MAU_ATENDIMENTO, ESTRUTURA_OPERACAO, PRODUTO_ESTRAGADO, PRODUTO_AVARIA, PRODUTO_EM_FALTA, PRECO_PRODUTO. Rejeita FALAR_RH, FALAR_DP, CURRICULO, FORNECEDOR, INFORMACAO_LOJA.
201Chamado criado com protocolo
409Já existe chamado não cancelado no atendimento
400Status, loja, categoria ou validação inválidos
PATCH /sac-cases/:id/status Transição de status do chamado 
{ "status": "sent_to_dkw" }

Valores permitidos: sent_to_dkw, in_resolution, resolved.

  • sent_to_dkw — idempotente, preserva sentToDkwAt
  • in_resolution — idempotente
  • resolved — move atendimento para pesquisa_satisfacao, preenche satisfactionRequestedAt
Marcar markAsSentToDkw: true no POST /sac-cases já cria como sent_to_dkw.
POST /sac-cases/:id/cancel Cancelar chamado

Apenas antes de resolved.

{
  "caseCancellationReason": "wrong_routing",
  "lastSummary": "Opcional para explicar a correcao.",
  "returnAttendanceToCollectingData": true
}

Motivos: operational_duplicate, created_by_mistake, wrong_routing, other.

returnAttendanceToCollectingData: true só para created_by_mistake ou wrong_routing. Se não retornar para coleta e não restar demanda útil, o atendimento é cancelado automaticamente.

POST /sac-cases/:id/protocol-sent Marcar protocolo enviado ao cliente 
{ "sentAt": "2026-05-24T12:00:00.000Z" }

sentAt opcional — usa horário do backend se omitido. Idempotente: nunca sobrescreve o primeiro timestamp.

GET /sac-cases/:id Detalhe do chamado

Chamado completo + atendimento resumido (attendanceId, status, externalConversationId, createdAt, closedAt).

GET /sac-cases/by-protocol/:protocol Buscar por protocolo

Case-insensitive, trim de espaços, exige hífens. Encontra chamados cancelados. Inclui replacedBy quando houver substituto.

GET /sac-cases/by-protocol/SAC-20260524-000001
GET /sac-cases (sem id) retorna 404 — não existe listagem global no MVP.
POST /sac-media Registrar mídia (metadados)

Idempotente por storageProvider + externalMediaId. Não baixa arquivo — só referência. Nunca cria chamado sozinha.

{
  "attendanceId": "attendance-id",
  "externalConversationId": "dkw-conversation-id",
  "caseId": null,
  "externalMediaId": "dkw-media-id",
  "storageProvider": "external",
  "sourceUrl": "https://example.com/media.jpg",
  "mimeType": "image/jpeg",
  "size": 123456,
  "purpose": "productImage"
}
CampoRegra
attendanceId ou externalConversationIdPelo menos um obrigatório
caseIdOpcional; se informado, deve pertencer ao atendimento
storageProviderPadrão external
purposeproductImage, receiptImage, storePhoto, other
201Nova mídia — created: true
200Reutilizada — created: false
400Atendimento em pesquisa_satisfacao ou validação
